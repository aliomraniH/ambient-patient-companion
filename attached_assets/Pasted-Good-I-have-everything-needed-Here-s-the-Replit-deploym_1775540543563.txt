### What changed (commit `f6047ab`)

Three files were modified to fix the ingestion pipeline — 3 bugs, +152/-42 lines:

**1. `mcp-server/skills/ingestion_tools.py` (main fix)**
Previously, `ingest_from_healthex()` short-circuited all text payloads (Format A/B/C) with `records_written: 0` — caching raw text but never parsing it. Now the text-payload branch calls `adaptive_parse()` (which routes to the existing format_b_parser, format_a_parser, etc.), maps the output through a new `_native_to_warehouse_rows()` helper, and feeds the rows into the existing per-table INSERT loop. Two new module-level helpers were added:
- `_parse_lab_value()` — extracts numeric values from lab strings like `"34.0-34.9"` or `">60"`
- `_native_to_warehouse_rows()` — maps parser native dicts to the warehouse schema (`biometric_readings`, `patient_conditions`, `clinical_events`, `patient_medications`)

**2. `server/deliberation/json_utils.py`**
Added `safe_json_loads()` — strips markdown fences before `json.loads()`, raises `ValueError` with a preview on failure instead of raw `JSONDecodeError`.

**3. `server/deliberation/synthesizer.py`**
Replaced bare `json.loads(raw)` at the synthesizer output with `safe_json_loads(raw)` to prevent crashes when Claude wraps its response in markdown fences.
