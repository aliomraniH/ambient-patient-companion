Context

plan contains several factual errors about how the codebase works. This corrected
plan preserves the goals while targeting the right code.

Phase 1 -- SDOH-First Serialization for LLM Context
Goal: Place SDOH flags before physiological data in context sent to LLMs, so
social determinants frame interpretation before lab values anchor reasoning.
Current state: PatientContextPackage (schemas.py:14-65) serializes via
model_dump_json() with SDOH at position ~13 (after conditions/meds/labs/vitals
at positions 8-12). The serialization is called in analyst.py:52,86 and
critic.py:60,89.
Changes
server/deliberation/schemas.py -- Add serialize_for_llm() method to
PatientContextPackage after line 65 (before DeliberationRequest at line 68):
pythondef serialize_for_llm(self) -> str:
    """Serialize with SDOH-first ordering for primacy bias mitigation."""
    import json
    raw = self.model_dump()
    ordered = {}
    # 1. Demographics
    for k in ("patient_id", "patient_name", "age", "sex", "mrn",
              "primary_provider", "practice"):
        if k in raw:
            ordered[k] = raw[k]
    # 2. SDOH context first (structural frame before clinical data)
    ordered["sdoh_flags"] = raw.get("sdoh_flags", [])
    # 3. Clinical state
    for k in ("active_conditions", "current_medications", "recent_labs",
              "vital_trends", "care_gaps"):
        if k in raw:
            ordered[k] = raw[k]
    # 4. Knowledge, guidelines, temporal, notes
    for k in ("prior_patient_knowledge", "applicable_guidelines",
              "upcoming_appointments", "days_since_last_encounter",
              "deliberation_trigger", "data_inventory",
              "clinical_notes", "available_media"):
        if k in raw:
            ordered[k] = raw[k]
    return json.dumps(ordered, indent=2, default=str)

Phase 4 -- Categorization Instruction in Critic Revision Prompt
Goal: Force structured "reasoning reset" between critique rounds by
requiring models to categorize findings before revising.
Current state: Revision prompt in critic.py:79-110 instructs models to
revise but doesn't require structured categorization of their reasoning state.
Why NOT orchestrate_refresh: orchestrate_refresh (ingestion_tools.py:1130)
is a deterministic Python pipeline. It calls HTTP endpoints and functions
sequentially. There is no LLM conversation context to inject prompts into.

revision prompt (after line 94, before line 96) with:

    UNCERTAIN (insufficient data), NEW (emerged from critique).
  Address CHALLENGED items first. Include the category in raw_revision.
This is a prompt text change only. No signature changes. The categorization
