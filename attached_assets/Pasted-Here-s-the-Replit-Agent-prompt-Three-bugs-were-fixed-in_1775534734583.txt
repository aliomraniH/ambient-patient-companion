## What Changed (branch: `claude/fix-healthex-logging-TElYa`)

### File 1: `mcp-server/transforms/fhir_to_schema.py`

**Fix A — Line 292:** `transform_by_type()` now passes `source` as the third argument to transform functions:
```python
transformed = fn([resource], patient_id, source)
```
Previously it was `fn([resource], patient_id)` — every record got `data_source="synthea"` (the default) and relied on a fragile post-hoc override on line 298.

**Fix B — Lines 206-217:** `transform_encounters()` now guards `r.get("type")` with `isinstance(..., list)` before indexing. When `type` is a raw string (e.g. `"encounter"`), it wraps it as `{"display": raw_type}` instead of crashing on `.get("coding")`. When `type` is a list of FHIR CodeableConcepts, it also checks `isinstance(type_list, dict)` before calling `.get()`.

### File 2: `ingestion/adapters/healthex/parsers/format_b_parser.py`

**Fix C — Encounters support added to Format B parser:**
- `_default_headers()`: added `"encounters"` key with columns `["Date", "Type", "Period", "Description", "Provider", "Status", "Location", "Code"]`
- `_build_col_dict_map()`: added `"Type": "C"` and `"Location": "C"` to known column-to-dictionary mappings
- `_to_native()`: added `elif resource_type == "encounters"` branch that maps to `{"type", "date", "description", "provider", "status"}` — matching what `_healthex_native_to_fhir_encounters()` in `server/mcp_server.py` expects
- `_deduplicate()`: added `"encounters": ("type", "date")` dedup key
