# Claude Code: Fix Lab 0.0 Placeholder Values (Run 14)

## Root Cause — Two-Level Bug, Precisely Diagnosed

The deliberation engine correctly self-diagnosed this in Run 13:
> "All recent labs (2025-07-11) report value=0.0 with missing units/flags;
>  data_inventory summary contradicts with A1c 4.8%, glucose 98 mg/dL"

The contradiction exists because the `insights_summary` is generated from the
ingestion plan's LLM planner (which reads the raw text and summarizes it
correctly), while the DB rows come from `format_a_parser.py` which parses the
text into dicts — and gets the field names wrong.

**Bug 1 — Wrong field mapping in format_a_parser.py labs section.**

The Format A LABS line format is:
```
"Hemoglobin A1c:4.8 %(ref:<5.7) 2025-07-11@Stanford Health Care...[totalrecords:9]"
"Glucose, Ser/Plas:98 mg/dL(ref:70-100 mg/dL) 2025-07-11@Stanford...[OutOfRange:5]"
"Narrative:Falsely decreased cholesterol... 2025-07-11@Stanford..."  ← skip this
"ECG Impression:- NORMAL ECG... 2017-03-31@Stanford..."              ← skip this
```

The current parser regex captures:
```python
m = re.match(r'(.+?):(.+?)\s+(\d{4}-\d{2}-\d{2})@(.+?)(?:\[|$)', item.strip())
# -> name="Hemoglobin A1c", value_unit="4.8 %(ref:<5.7)", date="2025-07-11"
```

Then writes the row as:
```python
{"test_name": "Hemoglobin A1c", "raw_value": "4.8 %(ref:<5.7)", "date": "2025-07-11"}
```

But the `patient_health_records` warehouse table expects:
```
result_value   VARCHAR  — the numeric part: "4.8"
result_unit    VARCHAR  — the unit part: "%"
ref_range      VARCHAR  — the reference range: "<5.7"
effective_date DATE     — not "date"
flag           VARCHAR  — "" or "out_of_range"
loinc_code     VARCHAR  — not extracted at all from Format A
```

Because the parser writes `date` but the table column is `effective_date`, the
upsert either silently ignores it or maps to NULL. Similarly `result_value` is
NULL/0.0 because it was never extracted — only `raw_value` was set, which is
not a column in the table.

**Bug 2 — write_and_verify conflict key uses "date", table uses "effective_date".**

The upsert conflict keys for labs are `["patient_id", "test_name", "date"]` but
the actual unique constraint is on `(patient_id, test_name, effective_date)`.
This means every upsert hits "no conflict key found" and either inserts a new
row or fails silently, writing 0.0/NULL into `result_value`.

## Step 2 — Rewrite the Labs Section of `format_a_parser.py`

Replace the existing labs block entirely. The key changes:

1. Split `raw_value` into `result_value` + `result_unit` + `ref_range`
2. Rename `date` → `effective_date`
3. Skip narrative/free-text rows (no numeric value)
4. Handle the `[OutOfRange]` flag correctly

```python
import re

# Rows whose test name starts with these are pure narrative — skip them
NARRATIVE_PREFIXES = {
    "Narrative", "Interpretation", "Comment", "Quantiferon Criteria",
    "QuantiFERON Incubation", "Alb/Creat Interp", "ECG Impression",
    "Fasting",  # boolean field, not a quantitative result
}


def _parse_lab_value(raw: str) -> tuple[str, str, str]:
    """
    Split a lab value+unit+ref string into (result_value, result_unit, ref_range).

    Input examples:
      "4.8 %(ref:<5.7)"          -> ("4.8", "%", "<5.7")
      "98 mg/dL(ref:70-100)"     -> ("98", "mg/dL", "70-100")
      "Negative"                 -> ("Negative", "", "")
      "9.1 mIU/mL"               -> ("9.1", "mIU/mL", "")
      "108 mL/min/1.73 m2(ref:>60)" -> ("108", "mL/min/1.73 m2", ">60")
      "<30"                      -> ("<30", "", "")
      "Non Reactive"             -> ("Non Reactive", "", "")
    """
    raw = raw.strip()

    # Extract ref range if present: (ref:...) or (ref:<...) etc.
    ref_range = ""
    ref_match = re.search(r'\(ref:([^)]+)\)', raw)
    if ref_match:
        ref_range = ref_match.group(1).strip()
        raw = raw[:ref_match.start()].strip()  # remove ref block from raw

    # Now raw is like "4.8 %" or "98 mg/dL" or "Negative" or "108 mL/min/1.73 m2"
    # Split on first whitespace after numeric prefix
    # Numeric prefix: digits, optional decimal, optional leading < or >
    num_match = re.match(r'^([<>]?[\d.]+)\s*(.*)', raw)
    if num_match:
        result_value = num_match.group(1).strip()
        result_unit = num_match.group(2).strip()
    else:
        # Non-numeric result (Negative, Non Reactive, Comment, etc.)
        result_value = raw
        result_unit = ""

    return result_value, result_unit, ref_range


def _is_numeric(value: str) -> bool:
    """True if value is a numeric result (possibly prefixed with < or >)."""
    return bool(re.match(r'^[<>]?[\d.]+$', value.strip()))


def parse_labs_from_summary(raw: str) -> list[dict]:
    """
    Parse the LABS section from a Format A plain text summary.

    Format A LABS line:
      "Test Name:value unit(ref:range) date@Provider[totalrecords:N,OutOfRange:M]"
    Items are pipe-separated (| ).
    """
    rows = []

    # Extract the LABS section — everything between LABS(N): and next section header
    match = re.search(
        r'LABS\(\d+\):\s*(.+?)(?:\nALLERGIES|\nIMMUNIZATIONS|\nCLINICAL VISITS|$)',
        raw,
        re.DOTALL
    )
    if not match:
        return rows

    items = match.group(1).strip().split(" | ")

    for item in items:
        item = item.strip()
        if not item:
            continue

        # Outer parse: "Test Name:value_block date@Provider[...]"
        # The date is YYYY-MM-DD, optionally preceded by (ref:...) block
        outer = re.match(
            r'(.+?):'           # test name (greedy up to first colon)
            r'(.+?)'            # value+unit+ref block (non-greedy)
            r'\s+(\d{4}-\d{2}-\d{2})'  # date
            r'@(.+?)'           # provider
            r'(?:\[.*)?$',      # optional [totalrecords:...] suffix
            item,
            re.DOTALL
        )
        if not outer:
            continue

        test_name = outer.group(1).strip()
        raw_value_block = outer.group(2).strip()
        effective_date = outer.group(3).strip()
        # provider = outer.group(4) — available but not needed for now

        # Skip known narrative / non-quantitative rows
        if any(test_name.startswith(p) for p in NARRATIVE_PREFIXES):
            continue

        # Skip rows where the value block is a long sentence (free text)
        # Heuristic: if raw_value_block contains more than 8 words, it's narrative
        if len(raw_value_block.split()) > 8:
            continue

        # Parse the value block
        result_value, result_unit, ref_range = _parse_lab_value(raw_value_block)

        # Determine out_of_range flag from the [OutOfRange] suffix
        flag = "out_of_range" if "[OutOfRange]" in item else ""

        rows.append({
            "test_name":     test_name,
            "result_value":  result_value,   # ← was "raw_value", now correct field
            "result_unit":   result_unit,    # ← new — was missing entirely
            "ref_range":     ref_range,      # ← new — was missing entirely
            "effective_date": effective_date, # ← was "date", now correct field
            "flag":          flag,
            "loinc_code":    "",             # Format A doesn't include LOINC
            "source":        "healthex_summary",
        })

    return rows
```

## What Changes End-to-End

### Before (broken):
```
Format A text: "Hemoglobin A1c:4.8 %(ref:<5.7) 2025-07-11@Stanford..."
         ↓ parser
Row written:   {"test_name": "Hemoglobin A1c", "raw_value": "4.8 %(ref:<5.7)", "date": "2025-07-11"}
         ↓ upsert (field name "date" not a column, "raw_value" not a column)
DB row:        {"test_name": "Hemoglobin A1c", "result_value": 0.0, "effective_date": NULL}
         ↓ Tier 1 context loader
Context:       {"test": "Hemoglobin A1c", "value": "0.0", "unit": ""}
         ↓ deliberation engine
Output:        "Current labs show 0.0 placeholders — cannot assess metabolic status"
```

### After (fixed):
```
Format A text: "Hemoglobin A1c:4.8 %(ref:<5.7) 2025-07-11@Stanford..."
         ↓ parse_labs_from_summary()
Row written:   {"test_name": "Hemoglobin A1c", "result_value": "4.8", "result_unit": "%",
                "ref_range": "<5.7", "effective_date": "2025-07-11", "flag": ""}
         ↓ upsert with correct conflict keys: (patient_id, test_name, effective_date)
DB row:        {"test_name": "Hemoglobin A1c", "result_value": "4.8", "result_unit": "%",
                "ref_range": "<5.7", "effective_date": "2025-07-11"}
         ↓ Tier 1 context loader
Context:       {"test": "Hemoglobin A1c", "value": "4.8", "unit": "%", "date": "2025-07-11"}
         ↓ deliberation engine
Output:        "A1c 4.8% (below prediabetes threshold) — glycemic control excellent"
```
