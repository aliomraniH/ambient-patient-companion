# Replit Agent: Deploy Clinical Data Storage Fix (Migration 005)

## What to do

Pull branch `claude/fix-clinical-data-storage-AXjlb` and deploy the clinical data storage fix. This branch has two commits:
- `a94433d` — Code changes: widens VARCHAR(20) columns, adds 11 structured lab result columns, fixes 3 write paths and 1 read path
- `e27039a` — Deployment guide + edge-case test data script

### The problem being fixed

`VARCHAR(20)` on `biometric_readings.unit` was silently truncating clinical data. Real medical units like `%{HemoglobinSaturation}` (23 chars) and reference ranges like `Male: 13.5-17.5 g/dL; Female: 12.0-16.0 g/dL` (45 chars) exceed the 20-char limit. Non-numeric lab results like "Reactive (Confirmed)" were being crammed into the unit field because there was no proper text column. `DOUBLE PRECISION` for `value` introduced floating-point imprecision for clinical threshold comparisons (e.g., 126 mg/dL glucose cutoff).
